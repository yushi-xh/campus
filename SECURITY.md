# 安全政策

本项目重视用户的数据与系统安全，遵循以下政策：

## 敏感信息管理
- 不要将 `login_config.json` 提交到公共仓库；使用 `login_config.json.example` 作为模板。
- 配置文件可能包含明文密码。请设置合理文件权限，避免非授权访问。
- 不在代码中硬编码账号与密码，不使用示例账号进行实际登录。

## 平台与网络安全
- 程序仅在本地运行，不会向第三方服务上传任何数据。
- 所有网络请求仅用于校园网认证流程。
- 建议在受信任网络环境中使用。

## 版本发布安全
- 发布可执行文件时不包含个人配置文件。
- 在变更日志中明确安全相关变更（如监控间隔、注册表清理机制）。

## 报告安全问题
如发现潜在安全问题或漏洞，请通过 Issue 或私信渠道与维护者联系。我们将及时修复并发布更新。
